# Contributing:

Any contribution is very welcome if it fits within the project plans.
This means that **you first need to know the project**, see what is already done or what is planned and integrate into that.
Features to solve your specific use case are ussually not very welcome.

You can use any tools to help you, but AI slop is not welcome.
And please don't promote your favourite model.

To ensure that we are in synchrony and don't waste any time in vain, I propose this approach:

1. Propose.
2. Discuss.
3. Make a decision.
4. Work.
5. PR.
